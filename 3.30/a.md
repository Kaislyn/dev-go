Apifox 新手入门教程（仅讲核心操作，全程手把手）

适用人群：完全零基础，不知道点击哪里、怎么新建项目/接口、怎么测试的新手；重点适配你测试 JWT 接口的场景，全程贴合你的 Go 后端接口测试需求。

核心目标：学会 3 件事——新建项目、新建接口、测试接口（含 JWT 令牌配置），其他复杂功能暂时不讲，避免混淆。

一、前期准备（必做）

1. 打开 Apifox 软件（电脑已安装，双击图标启动，无需登录也可使用本地项目）；

2. 启动后，默认进入「首页」，看不到多余内容，重点看「新建项目」按钮（核心入口）。

二、第一步：新建项目（拿到新项目必做）

目的：把测试的所有接口，都放在一个项目里，方便管理（比如你的 Go-JWT 接口，单独建一个项目）。

1. 点击首页最显眼的 「新建项目」按钮（红色/蓝色，位置在页面中间，一眼就能看到）；

2. 弹出新建项目窗口，只填 2 个关键信息（其他默认，不用改）：
        

  - 项目名称：随便填，贴合你的需求（比如「Go-JWT 接口测试」）；

  - 项目描述：可选，可填「测试 Go 后端 JWT 相关接口」，方便后续回忆；

3. 点击窗口右下角的 「创建」按钮，等待 1 秒，自动进入项目内部界面（新手不用管其他弹窗，直接关闭即可）。

提示：进入项目后，左侧是「接口管理」（核心区域），中间是工作区，右侧是环境配置（暂时不用管），全程重点关注左侧和中间。

三、第二步：新建接口（核心操作，重点学）

目的：把你 Go 后端写的接口（比如生成 Token、解析 Token、获取用户信息），添加到 Apifox 里，才能测试。

以你常用的「获取用户信息接口（/api/profile）」为例，全程演示：

1. 点击左侧导航栏的 「接口管理」（第一个选项，默认展开）；

2. 点击「接口管理」右侧的 「+ 新建接口」按钮（红色小加号，在接口管理标题的右边，很显眼）；

3. 弹出接口编辑窗口，重点填写「基本信息」（其他标签暂时不用管）：
        

  - 接口名称：填接口的用途（比如「获取用户信息」）；

  - 请求方法：点击下拉框，选择你接口的方法（比如你的接口是 GET，就选 GET；如果是 POST 就选 POST，和你 Go 代码里的方法一致）；

  - 接口路径：填你 Go 代码里的接口地址（比如 /api/profile，注意：不用填 localhost:8080，后面统一配置）；

4. 配置「请求头」（测试 JWT 接口必做！因为需要携带 Token）：
        

  - 点击窗口上方的「请求头」标签（在「基本信息」右边）；

  - 点击「请求头」里的 「+ 添加」按钮；

  - 会出现 2 个输入框，分别填：
            

    - Key：填 Authorization（固定写法，JWT 令牌必须放在这个请求头里）；

    - Value：先空着，测试的时候再填「Bearer + 你的 JWT 令牌」（比如 Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...，注意 Bearer 后面有一个空格）；

5. 保存接口：点击窗口右上角的 「保存」按钮（绿色/蓝色，点击后，接口就新建完成了，会出现在左侧接口管理的列表里）。

注意：新建接口时，只填「基本信息」和「请求头」就够了，其他标签（请求参数、响应示例）暂时不用管，不影响测试。

四、第三步：测试接口（最核心，新手必会）

目的：模拟前端发送请求，验证你 Go 后端的接口是否能正常返回数据，尤其是 JWT 令牌是否有效。

前提：你的 Go 后端代码已经启动（能正常生成 Token、接收请求），并且已经拿到生成的 JWT 令牌（从 Go 控制台复制）。

1. 找到要测试的接口：在左侧「接口管理」列表里，点击你刚才新建的接口（比如「获取用户信息」）；

2. 进入接口测试界面，点击上方的 「运行」标签页（在「修改文档」右边，新手容易找不到，重点看）；

3. 配置环境（关键！避免每次填 localhost:8080）：
        

  - 点击界面右上角的「环境」下拉框（默认是「无环境」），选择「新建环境」；

  - 环境名称填「本地测试环境」，环境地址填 http://localhost:8080（你的 Go 后端启动地址，必须和代码里的一致）；

  - 点击「保存」，然后环境下拉框选择「本地测试环境」（这样接口路径会自动拼接成http://localhost:8080/api/profile，不用手动填）。

4. 填写 JWT 令牌：
        

  - 在「运行」标签页的「请求头」里，找到刚才填的「Authorization」，在 Value 里粘贴你的 JWT 令牌（格式：Bearer 你的令牌字符串，切记 Bearer 后面有空格）；

5. 发送请求：点击界面右上角的 「发送」按钮（红色/蓝色，最大的那个按钮）；

6. 查看测试结果（重点看 2 个地方）：

  - 状态码：在响应区域的最上方，显示 200 → 测试成功（说明 Token 有效，接口正常）；显示 401 → Token 无效/过期（重新生成 Token 再试）；显示 404 → 接口地址填错了，检查路径；

  - 响应内容：状态码 200 时，下方会显示你 Go 后端返回的用户信息（比如 UserID、UserName、Role），和你代码里的返回一致，就说明测试通过。

五、新手常见操作补充（必看）

1. 新建多个接口（比如生成 Token 接口）

和新建「获取用户信息接口」步骤完全一样：左侧接口管理 → + 新建接口 → 填基本信息（名称、请求方法、路径）→ 按需配置请求头/请求体 → 保存 → 运行测试。

比如「生成 Token 接口」（POST 方法，路径 /api/login）：请求方法选 POST，请求体里填账号密码（如果你的接口需要），其他步骤一致。

2. 测试失败排查（新手最常遇到）

- 点击「发送」后没反应：检查 Go 后端是否启动，本地环境地址是否正确（localhost:8080）；

- 状态码 401：Token 填错了、过期了，或者格式不对（没加 Bearer + 空格），重新复制 Go 控制台生成的 Token；

- 状态码 404：接口路径填错了（比如多写了一个 /，或者少写了 api），和你 Go 代码里的路由保持一致；

- 状态码 500：后端代码有问题（比如 panic），去 Go 控制台看错误信息，修复后再测试。

3. 常用小技巧

- 复制 Token：Go 控制台生成 Token 后，选中复制，直接粘贴到 Apifox 的 Authorization Value 里，注意去掉多余空格；

- 保存测试用例：测试成功后，点击「运行」标签页里的「保存为用例」，下次测试不用重新填 Token，直接点击用例就能测试；

- 清空请求：如果测试失败，点击「运行」标签页里的「重置」按钮，重新填写参数再试。

六、终极总结（新手背会这 3 步）

1. 新建项目：首页 → 新建项目 → 填名称 → 创建；

2. 新建接口：接口管理 → + 新建接口 → 填基本信息 + 请求头 → 保存；

3. 测试接口：选中接口 → 运行 → 配置本地环境 → 填 Token → 发送 → 看状态码和响应。

你现在就可以跟着操作：先新建项目，再新建你 Go 里的「生成 Token 接口」和「获取用户信息接口」，一步步测试，遇到卡壳的地方，对照教程再检查一遍，肯定能成功！
