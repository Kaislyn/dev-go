七、补充：Redis 简单介绍（贴合你的 Go-JWT 场景）
Redis 是一款 高性能的开源内存数据库，核心特点是“数据存放在内存中”，读写速度极快，常用来解决你之前了解的「传统 Session 共享痛点」，也是 Go 后端开发中常用的工具，和 JWT 搭配使用场景极多。

1. 核心作用（新手重点记）

- 解决 Session 共享：多服务器集群时，将用户 Session 存到 Redis 中，所有服务器都能读取，避免用户重新登录（对应你之前学的传统 Session 痛点）；

- 缓存 JWT 令牌：可将生成的 JWT 令牌缓存到 Redis，方便快速校验、手动过期（比如用户登出时，直接删除 Redis 中的令牌，实现强制登出）；

- 缓存高频数据：比如用户信息、接口返回结果，减少数据库查询压力，提升接口响应速度。

2. 关键特点（新手易懂版）

- 快：内存读写，速度是传统数据库的几十到上百倍；

- 轻量：安装简单、配置简洁，新手容易上手；

- 多功能：支持字符串、哈希、列表等多种数据类型，适配多种开发场景；

- 可持久化：支持将内存中的数据保存到磁盘，避免重启后数据丢失。

3. 和你的 Go-JWT 关联

你目前测试的 JWT 接口，若后续扩展为多服务器部署，就可以用 Redis 缓存 JWT 令牌和用户 Session；本地测试阶段可暂时不用，但了解 Redis 能帮你理解大厂后端的真实架构，为后续学习铺垫。

### 为什么有了 MySQL，还要学 Redis？

在面试中，如果面试官问：“你的项目为什么用 Redis？”，最标准的大厂回答是：**为了解决 MySQL 的性能瓶颈。**

1. **MySQL 的软肋（磁盘 IO）**：MySQL 的数据是存在硬盘里的。哪怕你用了最快的固态硬盘，当遇到“双十一秒杀”或者“微博热搜”这种瞬间涌入几万个请求的情况，MySQL 每次都要去硬盘里翻数据，大概率会直接卡死崩溃（业内叫“宕机”）。
2. **Redis 的降维打击（内存操作）**：Redis 是一个**基于内存**的键值对（Key-Value）数据库。内存的读写速度是硬盘的几万倍！



大厂的经典架构是：**Redis 挡在 MySQL 前面做保镖。** 前端发来请求，后端先去查 Redis。如果有数据（缓存命中），直接以毫秒级的速度返回；如果没有，再去查 MySQL，查完之后顺手把数据塞进 Redis 里，这样下一个用户来查的时候就极快了。

### 第一步：在你的 Mac 上安装并启动 Redis

因为你的 Mac 是一台干干净净的新机器，并且我们已经配置好了强大的 Homebrew，安装 Redis 只需要动动手指。

打开你的终端，按顺序执行以下三条命令：

**1. 下载并安装 Redis：**
```bash
brew install redis
```

**2. 启动 Redis 服务（让它在后台默默运行）：**
```bash
brew services start redis
```
*(注意：以后只要你不执行 `brew services stop redis`，每次你的 Mac 开机，Redis 都会自动在后台为你准备就绪。)*

**3. 测试是否启动成功：**
Redis 自带了一个极简的命令行客户端，我们在终端里输入：
```bash
redis-cli ping
```
如果你看到终端里干净利落地弹出一个 **`PONG`**，就说明 Redis 已经成功在你的本地 `6379` 端口跑起来了！
