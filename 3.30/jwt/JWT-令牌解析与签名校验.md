# JWT 令牌解析与签名校验 
## 一、核心终极结论
1. **重新签名 = 原始签名**
   代表：令牌**未被篡改、是服务器合法签发**，校验通过，可正常使用。
2. **重新签名 ≠ 原始签名**
   代表：令牌**被恶意篡改/是伪造的**，直接失效报废，服务器拒绝校验。

---

## 二、JWT 解析校验完整执行流程
服务器接收到前端传递的 JWT 令牌后，会**严格执行以下5步校验**，缺一不可：
1. **拆分令牌**
   库自动将 JWT 按 `.` 分割为三部分：`Header（头部）` + `Payload（载荷）` + `原始签名（Signature）`。
2. **校验算法**
   验证 Header 中声明的签名算法，是否为服务器指定的 `HS256`（HMAC-SHA256），非指定算法直接判定非法。
3. **执行重新签名**
   服务器使用**自身密钥** + 拆分出的 `Header+Payload` + 指定算法，**重新计算生成一个新签名**。
4. **签名比对**
   将「重新生成的新签名」与「令牌自带的原始签名」做全等对比。
5. **最终判定**
   签名一致 → 合法；签名不一致/令牌过期/算法异常 → 非法。

---

## 三、重新签名核心定义与原理
### 1. 什么是重新签名？
服务器在校验令牌时，**复用签发时的算法和密钥**，对令牌中原有的 Header 和 Payload 重新计算一次签名，这个过程就是**重新签名**。

### 2. 重新签名公式（底层逻辑）
```go
新签名 = HMACSHA256( Base64UrlEncode(Header) + "." + Base64UrlEncode(Payload) , 服务器密钥 jwtKey )
```
(注：底层库会自动将 Header 和 Payload 进行 Base64Url 编码，并用 . 拼接后，再结合密钥进行哈希签名计算。)

### 3. 核心作用
唯一作用：**验证令牌的真实性和完整性**，防止令牌被篡改、伪造，是 JWT 安全体系的核心。

## 四、四大校验场景全罗列（覆盖所有情况）
### 场景1：正常合法令牌（用户正常登录，未修改）
- 触发条件：用户登录后，服务器签发正版令牌，前端原封不动传递给后端
- 校验流程：拆分令牌 → 算法校验通过 → 重新签名 = 原始签名 → 未过期
- 校验结果：✅ 校验通过
- 代码表现：`ParseToken` 无报错，成功解析出 `CustomClaims` 用户信息

### 场景2：令牌被恶意篡改（修改Payload用户ID/角色）
- 触发条件：人为修改 Payload 中的 `UserID`/`Role` 等字段
- 校验流程：拆分令牌 → 算法校验通过 → 重新签名 ≠ 原始签名
- 校验结果：❌ 校验失败，令牌直接报废
- 代码表现：`ParseToken` 返回签名校验错误

### 场景3：第三方伪造令牌（无服务器密钥，瞎编令牌）
- 触发条件：攻击者无服务器密钥，自行拼接伪造 JWT 令牌
- 校验流程：拆分令牌 → 算法校验通过 → 重新签名 ≠ 原始签名
- 校验结果：❌ 校验失败，伪造无效
- 代码表现：`ParseToken` 返回签名非法错误

### 场景4：令牌过期（超过设定的24小时有效期）
- 触发条件：令牌签发时间超过 `ExpiresAt` 设定的过期时间
- 校验流程：拆分令牌 → 算法校验通过 → 签名一致 → 令牌已过期
- 校验结果：❌ 校验失败，令牌失效
- 代码表现：`ParseToken` 返回令牌过期错误

## 五、关键疑问解答
### 1. 令牌被改废之后，重新登录有用吗？
**有用，且是唯一解决办法**
- 被篡改的令牌永久失效，服务器永远不会认可；
- 重新登录 → 服务器会**生成全新的正版JWT令牌**；
- 新令牌未被修改，重新签名与原始签名一致，可正常校验使用。

### 2. 每个用户令牌都不一样，为什么都能校验通过？
校验**不对比令牌本身是否重复**，只对比「重新签名」和「原始签名」是否一致。
只要是服务器用「指定算法+密钥」签发的令牌，无论用户是谁、何时登录，重新签名都能匹配。

### 3. 重新签名 = 重新生成令牌吗？
**完全不是**
- 生成令牌：服务器为用户签发身份凭证，返回给前端；
- 重新签名：仅在校验时计算签名，**不生成新令牌**，仅做合法性验证。

## 六、结合代码解析校验逻辑
```go
// 核心解析校验函数
func ParseToken(tokenString string) (*CustomClaims, error) {
	// 👇 该行自动完成：拆令牌+验算法+重新签名+签名比对
	token, err := jwt.ParseWithClaims(tokenString, &CustomClaims{}, func(token *jwt.Token) (interface{}, error) {
		// 校验算法：仅允许HMAC系列算法，防御恶意攻击
		if _, ok := token.Method.(*jwt.SigningMethodHMAC); !ok {
			return nil, fmt.Errorf("unexpected signing method: %v", token.Header["alg"])
		}
		// 返回服务器密钥，供库执行【重新签名】
		return jwtKey, nil
	})

	// 签名不匹配/过期/格式错误 → 直接返回错误
	if err != nil {
		return nil, err
	}

	// 签名一致+令牌有效 → 解析成功，返回用户载荷信息
	if claims, ok := token.Claims.(*CustomClaims); ok && token.Valid {
		return claims, nil
	}
	return nil, fmt.Errorf("invalid token")
}
```

## 七、终极记忆口诀
解析只做一件事：**重新算签名，对比原签名**
签名一致=合法，签名不一=报废
令牌改废不用慌，**重新登录拿新证**
校验不看旧与新，只认签名对不对
